# 雅江项目当前已接入数据集说明

本文档用于说明当前雅江项目中已经发现并初步确认可用的几个数据集，包括它们的组织方式、在当前 AEF 初版训练框架中的角色、数据形态，以及现阶段的使用建议。

---

## 1. 数据总览

当前在服务器目录 `/workspace/raw/yajiang/` 下已经存在以下数据源：

- `s2`
- `s1`
- `landsat`
- `dem`
- `worldcover`
- `modis_ndvi`
- `modis_lst`
- `dynamic_world`
- `jrc_water`

从目录结构来看，这些数据已经按 **patch** 组织，而不是散乱的原始整幅影像，这对后续训练非常有利。

---

## 2. 当前最适合用于训练闭环的数据组合

结合目前项目代码结构和真实数据检查结果，当前最适合用于第一阶段真实数据训练（v0.2-a）的组合是：

### 输入源
- `s2`
- `s1`

### 监督目标
- `dem`
- `worldcover`

### 暂时不优先接入
- `jrc_water`

### 暂时延后考虑
- `landsat`
- `modis_ndvi`
- `modis_lst`
- `dynamic_world`

原因是当前 `s2 + s1 -> dem + worldcover` 这条链最简单、最稳定、最适合先打通真实数据训练流程。

## 3. 各数据集详细说明

---

## 3.1 Sentinel-2 (`s2`)

### 数据角色
当前项目中，`s2` 作为 **主要输入源之一**。

### 数据组织方式
目录结构类似：

```text
/workspace/raw/yajiang/s2/
  patch_000000/
    2023Q1.tif
    2023Q2.tif
    ...
    2026Q1.tif
  patch_000001/
    2023Q1.tif
    ...
````

说明：

* 已按 patch 组织
* 每个 patch 下按季度存储
* 是一个带时间维度的时序输入源

### 数据形态

抽样检查结果表明：

* 波段数：`6`
* 尺寸：`128 × 128`
* 数据类型：`float64`
* 坐标系：`EPSG:32646`

### 当前使用建议

* 可以直接作为模型输入
* 与当前配置中 `s2: 6` 是一致的
* 后续建议在预处理阶段转换为 `float32`，减少存储和训练开销

### 当前意义

`s2` 提供的是多光谱光学信息，是当前模型最重要的观测输入之一。

## 3.3 DEM (`dem`)

### 数据角色

当前项目中，`dem` 作为 **连续值监督目标（target）** 使用，而不是主输入源。

### 数据组织方式

目录结构类似：

```text
/workspace/raw/yajiang/dem/
  patch_000000/
    static.tif
  patch_000001/
    static.tif
  ...
```

说明：

* 已按 patch 组织
* 每个 patch 只有一个静态文件
* 不带时间维度

### 数据形态

抽样检查结果表明：

* 波段数：`1`
* 尺寸：`43 × 43`
* 数据类型：`float32`
* 坐标系：`EPSG:32646`

### 当前使用建议

* 非常适合作为连续值重建目标
* 需要注意它的分辨率 **低于输入源**
* 在训练时应允许 target 分辨率与输入不同
* 适合使用 continuous loss（如 L1 / MSE）

### 当前意义

`dem` 提供地形结构信息，有助于模型学习地表物理结构。

---

## 3.4 ESA WorldCover (`worldcover`)

### 数据角色

当前项目中，`worldcover` 作为 **分类监督目标（categorical target）** 使用。

### 数据组织方式

目录结构类似：

```text
/workspace/raw/yajiang/worldcover/
  patch_000000/
    static.tif
  patch_000001/
    static.tif
  ...
```

说明：

* 已按 patch 组织
* 每个 patch 一个静态标签文件
* 不带时间维度

### 数据形态

抽样检查结果表明：

* 波段数：`1`
* 尺寸：`128 × 128`
* 数据类型：`uint8`
* 坐标系：`EPSG:32646`

### 类别情况

在前 20 个 patch 中观察到的唯一值包括：

```text
10, 30, 40, 50, 60, 70, 80, 100
```

说明：

* 它是一个单通道类别标签图
* 当前至少观测到 8 个类别
* 原始标签值不是从 `0` 开始连续编号

### 当前使用建议

* 很适合作为 v0.2-a 的首个分类监督目标
* **必须做类别重映射**
* 不建议直接把 `10/30/40/.../100` 直接送入交叉熵损失

### 推荐映射方式

建议在预处理或 `.tif -> .npy` 转换阶段就完成映射，例如：

```text
10  -> 0
30  -> 1
40  -> 2
50  -> 3
60  -> 4
70  -> 5
80  -> 6
100 -> 7
```

### 当前意义

`worldcover` 提供土地覆盖语义监督，是当前最适合先接入的分类 target。

说明：

* 已按 patch 组织
* 每个 patch 一个静态文件
* 不带时间维度

### 数据形态

抽样检查结果表明：

* 波段数：`1`
* 尺寸：`43 × 43`
* 数据类型：`int16`
* 坐标系：`EPSG:32646`

### 类别情况

在前 20 个 patch 中观察到的唯一值包括：

```text
-128, 2, 3, 4, 5, 6, 7, 9, 20, 21, 23, 27, 30, 33, 40
```

### 当前判断

这说明：

* `-128` 很可能是 `nodata` 或无效值
* 其余类别值也不是连续编号
* 当前它需要同时处理：

  * 类别重映射
  * 无效值 mask
  * 较低分辨率 target

### 当前使用建议

* 当前阶段 **不建议优先接入**
* 建议在 v0.2-a 先移除
* 等 `s2 + s1 -> dem + worldcover` 跑通后，再考虑把它加入 v0.2-b 或 v0.3

### 当前意义

`jrc_water` 具有一定价值，但接入复杂度高于 `worldcover`，更适合后续阶段接入。

## 4. 当前数据组织方式的优点

当前 `/workspace/raw/yajiang/` 这批数据有几个非常明显的优点：

### 4.1 已按 patch 对齐

不同源都使用统一的 patch id，例如：

```text
patch_000000
patch_000001
...
```

这意味着跨源配对相对容易。

### 4.2 时序源和静态源分工清晰

* `s1/s2`：时序输入
* `dem/worldcover/jrc_water`：静态 target

这很适合当前 AEF 初版结构。

### 4.3 空间参考一致

当前抽样检查中看到所有已检查 GeoTIFF 都是：

```text
EPSG:32646
```

说明坐标系是一致的，这对后续处理非常有利。

## 5. 当前阶段的使用建议

## 5.1 v0.2-a 推荐最小真实训练组合

### 输入

* `s2`
* `s1`

### target

* `dem`
* `worldcover`

### 暂时移除

* `jrc_water`

这是当前最稳的第一版真实数据闭环。

---

## 5.2 当前最推荐的数据处理方式

由于当前代码 `dataset.py` 默认支持的是：

* `.npy`
* `.npz`
* `.pt`

而已有真实数据是：

* `.tif`

因此当前最推荐的做法是：

1. 先从真实 GeoTIFF 中抽取少量 patch
2. 将 `s2/s1/dem/worldcover` 转换为 `.npy`
3. 对 `worldcover` 做类别重映射
4. 生成最小 manifest
5. 用 `train_with_manifest.py` 跑通第一版真实数据训练

---

## 5.3 不建议当前立即做的事情

当前阶段不建议优先做：

* 接入 `jrc_water`
* 接入 `landsat`
* 接入 `modis_ndvi`
* 接入 `modis_lst`
* 接入 `dynamic_world`
* 多卡训练
* 复杂加速优化

原因是当前最重要的目标是：

**先把最小真实数据训练闭环跑通。**

## 7. 一句话总结

当前雅江数据已经具备良好的 patch 化组织形式，最适合先用于第一版真实数据训练的组合是：

**输入：`s2 + s1`
目标：`dem + worldcover`**

其中：

* `s2` 和 `s1` 已与当前模型输入配置对齐
* `dem` 适合作为连续值 target
* `worldcover` 适合作为分类 target，但需要先做类别重映射
* `jrc_water` 暂时不建议优先接入，因为其包含无效值且需要额外类别处理

这为后续 v0.2 小规模真实数据单卡联调提供了良好的基础。
