# AlphaEarth Foundations 论文解读

论文：`2507.22291v2.pdf`  
标题：AlphaEarth Foundations: An embedding field model for accurate and efficient global mapping from sparse label data  
arXiv HTML：https://arxiv.org/html/2507.22291v2

## 1. 这篇论文想解决什么问题

遥感领域并不缺影像，真正稀缺的是高质量标签。比如要做土地覆盖、水体、农作物、树种、蒸散发等制图任务，卫星影像很多，但人工标注、地面观测和实测样本通常很少，而且分布不均匀。

传统做法通常是：

- 针对某一个任务设计特征，比如 NDVI、时序合成、CCDC、纹理特征；
- 用少量标签训练一个分类器或回归器；
- 再把模型外推到大区域。

问题是这些特征往往依赖具体任务、传感器和区域，泛化能力有限。论文提出的 AlphaEarth Foundations，简称 AEF，想做一件更通用的事：

> 先用大量多源遥感数据训练一个通用地表 embedding，再用少量标签在 embedding 上快速训练下游模型。

也就是说，它不是直接为某一个任务训练一个最终模型，而是训练一个可以服务很多制图任务的遥感基础表征。

## 2. 核心概念：Embedding Field

AEF 的输出不是 RGB 图，也不是某个单一类别图，而是一组空间连续的特征向量。论文把这种结果称为 embedding field。

可以理解为：

```text
每一个地理位置 / 像元
        ↓
对应一个低维向量
        ↓
这个向量编码了该位置的地表状态、时间变化、多源观测信息
```

论文发布的是年度全球 embedding field，从 2017 到 2024，每年一套。每个像元不是一个类别，而是一个可以用于下游任务的特征向量。

这和普通监督模型的区别很重要：

- 普通模型：输入影像，直接输出水体/土地覆盖/作物类型；
- AEF：输入多源时序观测，输出通用 embedding；
- 下游任务：用少量标签在 embedding 上训练 kNN、线性分类器或回归器。

## 3. 模型输入是什么

论文里的 AEF 使用多源遥感数据，包括：

- Sentinel-2 光学影像；
- Sentinel-1 SAR 雷达影像；
- Landsat 8/9 光学与热红外；
- PALSAR-2 雷达；
- ERA5-Land 气候数据；
- GEDI LiDAR；
- GRACE 重力场；
- Copernicus DEM；
- NLCD 土地覆盖；
- Wikipedia / GBIF 文本与物种记录。

但论文也指出，模型输入源不宜盲目越多越好。最终用于核心视频输入的主要是 Sentinel-2、Sentinel-1 和 Landsat，因为它们能比较稳定地支撑多源重建。

对我们当前项目来说，对应关系是：

```text
论文 AEF: Sentinel-2 + Sentinel-1 + Landsat 等多源输入
当前项目: Sentinel-2 + Sentinel-1
```

我们现在是一个区域级、简化版复现，不是完整全球 AEF。

## 4. 网络结构怎么理解

论文的整体结构可以概括为：

```text
多源多时相遥感输入
        ↓
各传感器 source encoder
        ↓
STP 主干网络
        ↓
时间条件 summary
        ↓
vMF bottleneck
        ↓
embedding field
        ↓
decoder 重建不同数据源
```

### 4.1 Source Encoder

不同传感器的波段数、观测方式不同，所以模型先用独立的 source encoder 把它们投影到同一个 latent space。

我们项目里对应的是：

```text
src/models/sensor_encoders.py
```

当前支持：

```text
s2: 6 通道
s1: 2 通道
```

### 4.2 STP：Space-Time-Precision block

STP 是论文的核心结构之一。它同时处理三类信息：

- Space：空间关系，比如一个 patch 内不同位置之间的联系；
- Time：时间变化，比如同一位置不同季节或年份的变化；
- Precision：局部细节，比如纹理、边界、局部形状。

我们项目里对应的是：

```text
src/models/blocks.py
```

当前实现里，每个 STP block 有三条路径：

```text
precision path: 卷积，保留局部细节
time path: 时间维度 self-attention
space path: 空间维度 self-attention
```

这就是为什么模型不是简单 CNN。它需要同时理解遥感影像的空间结构和时间变化。

### 4.3 Time-Conditional Summary

AEF 很强调时间。论文区分了两个概念：

- support period：输入观测实际覆盖的时间范围；
- valid period：希望模型总结的目标时间范围。

这样模型可以处理插值和外推。例如输入影像不一定完全覆盖目标年份，但模型仍然要给出某个时间窗口的地表 summary。

我们项目里也有类似设计：

```text
valid_start_ms
valid_end_ms
target_relative_time
```

对应代码：

```text
src/models/time_encoding.py
src/models/model.py
```

不过我们现在的时间建模还比论文简化很多。

### 4.4 vMF Bottleneck

论文把 embedding 看成球面上的向量，并通过 von Mises-Fisher 分布引入噪声瓶颈。这个设计有两个目的：

- 压缩信息，逼模型学通用表征，而不是死记输入；
- 让 embedding 空间更适合距离计算，比如 kNN 检索和少样本分类。

我们项目里对应的是：

```text
src/models/bottleneck.py
```

当前 v0.3c 配置：

```yaml
embedding_dim: 128
vmf_kappa: 2000.0
```

论文正式模型的 embedding 更强调紧凑性，发布数据经过 8-bit 量化后每个 embedding 存储开销很小。

## 5. 模型到底在学习什么

AEF 的关键不是“直接预测某一张标签图”，而是通过重建任务训练 embedding。

论文里的训练目标包括：

- reconstruction objective：从 embedding 重建不同数据源；
- batch uniformity objective：让 embedding 在空间中分布更均匀，避免塌缩；
- consistency objective：缺失部分输入时，embedding 仍然保持一致；
- text contrastive objective：让地理文本语义和遥感 embedding 对齐。

我们当前项目对应的是简化版本：

```text
输入:
  Sentinel-2
  Sentinel-1

重建目标:
  DEM
  WorldCover
  JRC Water

正则:
  uniformity
  variance
  decorrelation
  orthogonality
```

所以当前模型学习的是：

- 从 S2/S1 中识别地形相关模式，用于重建 DEM；
- 从 S2/S1 中识别地表覆盖类型，用于重建 WorldCover；
- 从 S2/S1 中识别水体相关特征，用于重建 JRC Water；
- 同时让中间 embedding 更稳定、更分散、更适合下游任务。

一句话概括：

> DEM、WorldCover、JRC Water 不是最终目的，而是训练 embedding 的老师。

## 6. 为什么要重建，而不是直接做下游任务

如果只训练一个水体分割模型，模型会专门优化“水体”这个任务。但如果以后要做土地覆盖、植被、变化检测、地形相关分析，就可能需要重新训练。

AEF 的想法是先训练一个基础 embedding，让它包含多种地表信息。之后下游任务只需要少量标签，就可以在 embedding 上训练轻量模型。

这对遥感很有意义，因为很多实际制图任务都有这个特点：

- 标签很少；
- 标签很贵；
- 区域差异大；
- 任务种类多；
- 需要快速迁移。

因此，重建任务的作用不是为了部署时复制 DEM 或 WorldCover，而是为了逼迫模型学到可迁移的遥感语义。

## 7. 论文怎么评测

论文没有只看重建 loss，而是重点评测 embedding 对下游任务的帮助。

评测任务包括：

- land cover classification；
- land use classification；
- crop type mapping；
- tree genera classification；
- oil palm classification；
- evapotranspiration regression；
- emissivity regression；
- land cover / land use change detection。

评测方式强调少样本场景：

```text
1-shot
10-shot
max-trial
```

迁移方法也比较轻量：

```text
kNN
linear probe
```

论文比较的 baseline 包括：

- XY 坐标；
- XYZ 坐标加高程；
- ImageNet ViT；
- Composites；
- CCDC；
- MOSAIKS；
- SatCLIP；
- Prithvi；
- Clay。

论文结论是：AEF 在多个任务上整体优于这些人工特征和已有学习特征，尤其在标签稀缺的真实制图场景中更有优势。

这给我们项目一个直接启发：

> 不应该只汇报训练 loss，还应该建立下游 baseline，看 AEF embedding 对真实任务有没有提升。

我们已经新增的 `src/eval/` 和 `scripts/eval_baselines.py` 就是在往这个方向走。

当前 yajiang-aef:

## 9. 当前项目与论文的主要差距

当前项目已经抓住了 AEF 的主干思想，但和论文完整版还有明显差距：

1. 数据源更少  
   论文使用多种光学、雷达、气候、地形、LiDAR、文本数据；我们目前主要是 S2/S1，加 DEM、WorldCover、JRC Water 作为目标。

2. 训练规模更小  
   论文是全球级训练，使用大规模设备和海量样本；我们是雅江区域 1708 个 patch。

3. 目标任务更少  
   论文重建多种观测源，并包含文本对齐；我们当前主要重建 3 个目标。

4. 一致性学习还不完整  
   论文有 teacher/student 缺帧一致性目标；我们当前还没有完整实现这一块。

5. 下游评测仍需加强  
   论文核心证据来自少样本下游评测；我们目前已有 baseline 框架，但还需要构建严格 train/test split 和真实下游任务。

## 10. 对我们下一步工作的建议

### 10.1 先把评测做扎实

不要只看训练重建图。建议建立固定评测协议：

- 固定 train/test patch split；
- 下游任务至少包括 `jrc_binary`、`worldcover`、`dem regression`；
- 比较 composite features、AEF embedding、随机投影、kNN、linear probe；
- 报告 balanced accuracy、IoU、F1、MAE/R2 等指标。

### 10.2 加入缺帧一致性训练

论文非常重视输入稀疏和缺失问题。可以增加一个 teacher/student consistency：

```text
teacher: 使用完整 S2/S1 时序
student: 随机丢帧或丢 S1
loss: 让两者 embedding 接近
```

这会让模型在真实遥感缺云、缺观测时更稳定。

### 10.3 增强时间建模

现在我们的时间使用还比较轻。后续可以让 target_relative_time 和 valid period 更真实地参与目标重建，比如针对年度、季度、季节性任务建立不同 summary period。

### 10.4 做部署侧 embedding 输出

论文真正发布的是 embedding field，而不是 decoder 的重建图。我们也可以增加一个正式导出流程：

```text
输入一个区域 patch
输出 embedding_map.npy 或 GeoTIFF
下游任务读取 embedding 做分类/回归
```

这样项目会更接近 AEF 的思想。

## 11. 最短总结

这篇论文的核心贡献是：

> 用多源遥感时序训练一个通用 embedding field，让少量标签也能完成高质量制图。

它的技术关键是：

- 多源输入编码；
- STP 时空融合；
- 时间条件 summary；
- vMF 信息瓶颈；
- 多源重建目标；
- embedding uniformity；
- 缺失输入一致性；
- 少样本下游评测。

我们当前项目已经实现了其中最重要的一条主线：

```text
S2/S1 输入
  → STP 融合
  → vMF embedding
  → DEM / WorldCover / JRC Water 重建
  → 下游 baseline 评测
```

后续重点不是单纯把重建图做漂亮，而是证明这个 embedding 在少样本下游任务上比传统特征更有用。
