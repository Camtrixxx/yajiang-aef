## 一、项目概述

**xuannv_embdding** 是 AlphaEarth Foundations (AEF) 的独立改进版，核心目标是：
- **解决嵌入坍缩** (Embedding Collapse)
- **提升时间敏感性** (Temporal Sensitivity)
- **使模型能够执行变化检测** (Change Detection)

项目基于哈尔滨 424 个 patch 的遥感数据（S2/S1/Landsat），在 8× Huawei Ascend 910B4 NPU 上进行训练。

### 关键指标定义

| 指标 | 含义 | 健康范围 | 异常信号 |
|------|------|----------|----------|
| `raw_unif` | 嵌入均匀性（越负越好） | -4.0 ~ -1.0 | > -0.5 表示坍缩 |
| `recon` | 重建损失 | < 0.3 | > 0.5 重建质量差 |
| `active_dims` | 活跃维度数（标准差>阈值） | > 120/128 | < 20 严重坍缩 |
| `AUC` | 变化检测准确率 | > 0.7 及格 | < 0.6 接近随机 |
| `cov` | 协方差正则值 | < 1.0 | > 5.0 严重失控 |

## 二、数据源问题

### 2.1 S2 云量筛选导致的冬季数据缺失

**问题描述**:
- S2 原始数据每 patch 约 180 帧，云筛选后仅剩 ~22 帧/patch
- 357 个月份 (7.4%) 全 cloudy，被迫保留最不清的一张
- 主要集中在 2025-01 (272 个 patch 受影响)

**影响**:
- 冬季月份可用帧数极少，时间序列不完整
- 模型难以学习冬季→夏季的季节变化模式
- 云 fallback 数据质量差，引入噪声

**解决方法**:
- 实施云筛选：`filter_cloudy_frames.py --max-per-month 2 --cloud-threshold 0.3`
- 全 cloudy 月份 fallback 保留最少云帧
- 代码优雅处理缺失数据（`source_input_mask[s_idx]=False`）

**效果**:
- ✅ 数据量从 29,707 帧 → 9,321 帧，质量提升
- ⚠️ 但冬季数据仍然稀缺，时间覆盖不均衡

---

### 2.2 Landsat 缺失 17 个 Patch

**问题描述**:
- 424 个 patch 中，17 个 patch 完全没有 Landsat 数据
- 原代码可能因此 crash

**解决方法**:
- 在 `dataset.py` 中设置 `source_input_mask[s_idx]=False`
- 模型自动忽略缺失源，不报错

**效果**:
- ✅ 训练稳定，无 crash

---

### 2.3 不同源时间不对齐

**问题描述**:
- S2 (5天重访)、S1 (12天)、Landsat (16天) 时间不对齐
- 同一天三种源几乎无重叠帧

**影响**:
- 多源融合效果受限
- 模型主要依赖 S2 数据

**解决方法**:
- 通过时间编码器处理不同时间戳
- 这是设计预期行为，未做额外处理

**效果**:
- ⚠️ 多源优势未充分发挥

---

### 2.4 静态/动态目标比例失衡

**问题描述**:

| 对比 | AEF 论文 | 我们的实现 |
|------|---------|-----------|
| 动态目标 | 7 个 (78%) | 3 个 (43%) |
| 静态目标 | 2 个 (22%) | 4 个 (57%) |

- 静态目标（DEM/WorldCover/Dynamic World/JRC Water）时间无关
- 静态目标重建"太容易"，模型可能学到"时间不重要"的 bias
- WorldCover 是 2021 年标签，数据是 2023-2025 年，存在时间错位

**影响**:
- 57% 的重建信号是"时间无关"的
- 模型倾向于学习时间不变表征
- 梯度信号被静态目标主导

**解决方法**:
- 实施源特定重建权重（DEM=0.3，动态目标=1.0）
- 条件解码器设计（decoder 学会忽略静态目标的时间编码）

**效果**:
- ⚠️ 部分缓解，但未根治

---

### 2.5 标注数据时间窗口不匹配

**问题描述**:
- 变化检测标注对应相邻月份（4-6月、6-8月等）
- 评估脚本最初使用固定半年窗口（2023-07~12 vs 2024-07~12）
- S2 数据 99%+ 为 2025 年帧，固定窗口内几乎无数据

**影响**:
- Few-Shot CD AUC 从 ~0.65 → ~0.50（接近随机）

**解决方法**:
- 按 shapefile 对应月份设置窗口：
  - `june.shp` → before=2025-04, after=2025-06
  - `aug.shp` → before=2025-06, after=2025-08

**效果**:
- ✅ AUC 恢复正常

## 三、模型训练核心问题

### 3.1 Embedding 坍缩（最严重问题）

**问题描述**:
模型输出的 embedding 向量聚集在狭小区域，丧失区分能力：
- `active_dims` 从 ~50 → 0
- `std_mean` 从 ~0.05 → ~0.01
- 所有输出几乎相同向量

**历史案例**:
- V1~V7：多次尝试未找到稳定方案
- V13 Round 1：5 个实验全部 10 epoch 内坍缩到 0 维
- V13 Round 2：Skip L2 实验全部失败

**根因分析**:

| # | 根因 | 说明 |
|---|------|------|
| 1 | **GMP 梯度屏障** | Global Mean Pooling 将 4096 空间位置平均为 1 向量，消灭空间多样性 |
| 2 | **Reconstruction Loss 主导** | recon=0.5 时，模型发现"坍缩 embedding"反而能更好重建 |
| 3 | **L2 Norm Jacobian 屏障** | L2 归一化在坍缩时梯度爆炸或消失，uniformity loss 无法传播 |
| 4 | **Consistency Loss 中 teacher detach** | 反向传播被截断，gradient 无法到达 encoder |
| 5 | **每源只加载 1 帧** | `valid_period` 时间窗口完全失效 |
| 6 | **Decoder 太强** | 4 层 Conv 直接复制输入，无需 meaningful embedding |

**解决方法与效果**:

| 方案 | 效果 | 说明 |
|------|------|------|
| `skip_l2_training=True` | ✅ 关键突破 | 训练时在 pre-norm 空间计算损失，绕过 L2 Jacobian 屏障 |
| Spatial VICReg (GMP前) | ✅ 有效 | 在 `[B×H×W, D]` 上计算 variance/covariance，保留空间多样性 |
| recon=0（纯发散） | ✅ 最佳 | 10 epoch 达 93/128 active，但无实际表征学习能力 |
| recon=0.1（分阶段） | ✅ 可用 | 从 recon=0 加载，20 epoch 达 108/128 + recon=0.23 |
| Raw Uniformity | ❌ 无效 | ExpN/O 加入后反而坍缩 |

**当前状态**: Round 7 所有实验 active=128/128（修正阈值后约 82-111/128），基本解决坍缩。

---

### 3.2 时间敏感性缺失（时间盲）

**问题描述**:
模型学到过度的时间不变性，无法区分不同时间的地表状态：
- V1: cosine similarity = 0.97（几乎完全相同）
- V12 Bare AUC = 0.499（≈随机猜测）
- V5 Raw Embedding AUC = 0.528
- changed/unchanged separation = 0.0017（健康值 >0.05）

**根因分析**:

| # | 根因 | 说明 |
|---|------|------|
| 1 | **训练方式：同月自重建** | 输入=目标同月，模型只需学空间重建，忽略时间 |
| 2 | **时序对比损失符号反了** | V2 早期 `L = -cos_sim/0.1`，优化器反而鼓励相同 embedding |
| 3 | **Temporal Loss 在 global mean 计算** | 空间信息丢失，pixel-level 变化被平均 |
| 4 | **Reconstruction 主导** | 权重=1.0，模型优先学习静态内容，时间变化被视为噪声 |
| 5 | **缺少 Batch Uniformity** | AEF 核心机制缺失，batch 内样本缺乏多样性 |
| 6 | **Skip L2 的不一致性** | 训练 skip L2 → 推理 L2，pre-norm 差异被压缩 |
| 7 | **数据量不足** | 424 patches vs AEF 的 840 万序列，变化样本 <1% |

**解决方法与效果**:

| 方案 | 效果 | 说明 |
|------|------|------|
| 双窗口时序对比损失 | ✅ 改善 | Before/After cosine distance 从 0.03 → 0.58（19 倍） |
| Pixel-Level Temporal Loss | ✅ 更强 | 避免"全局偏移"作弊，空间粒度更细 |
| Gap-Aware Temporal Loss | ✅ 合理 | 根据时间 gap 动态设定 target |
| Cross-Temporal 重建 | ⚠️ 尝试中 | 输入 month A，重建 month B（Round 2 方案） |
| 高时序权重 (temporal=0.5) | ⚠️ 双刃剑 | 下游分类提升 (+3.2%)，但 AUC 下降 (-0.016) |

**当前状态**: Round 4 baseline AUC = 0.6918（及格），但仍有提升空间。

---

### 3.3 Covariance 爆炸

**问题描述**:
VICReg 协方差项失控增长：
- Mini Batch: cov 从 1.05 → 22.51（22 倍增长）
- `covariance_weight=0.001` 太低，无法抑制

**根因**:
- covariance_weight 过低
- warmup 只有 3 epochs，学习率上升过快
- pre_norm_uniform_weight=0.5 不足

**解决方法**:
- 提高 covariance_weight 到 0.01 或 0.05
- 延长 warmup 到 10 epochs
- 使用 Spatial VICReg（在 GMP 前计算）

**效果**:
- ✅ Spatial VICReg + recon=0 时 cov 控制最好

---

### 3.4 Active Dimensions 阈值不准确

**问题描述**:
原代码硬编码阈值 `0.05`，但在 pre-norm 空间过于宽松：

| 空间 | 实际 std_mean | 阈值占比 | 评价 |
|------|-------------|----------|------|
| Pre-norm spatial | ~0.36 | 13.9% | 🔴 过于宽松 |
| Pre-norm inter | ~0.24 | 20.8% | 🔴 过于宽松 |
| L2 理论 | 0.088 | 56.6% | 🟡 合理 |

- 导致 `128/128` 是假象，实际活跃度约 82-111/128

**解决方法**:
- 新增 `active_dims_threshold` 配置参数
- Pre-norm 实验设 0.15，L2 对照组保持 0.05

**效果**:
- ✅ 更真实反映 embedding 活跃度

## 四、评估与下游任务问题

### 4.1 分类目标重建 Loss 的 one-hot 解码错误（致命 Bug）

**问题描述**:
`loops.py` 中分类目标（worldcover、dynamic_world）的重建 loss：
```python
# 错误代码
tgt_cls = tgt[batch_mask, 0].long()  # 取 one-hot 的第 0 通道
```
- 类别 0 被错误教导预测为类别 1
- 类别 1-N 全部被错误教导预测为类别 0
- worldcover + dynamic_world 的重建监督完全错乱

**影响**:
- 所有历史训练（V1-V7）都受此 bug 影响
- 但 bottleneck classification loss（weight=0.03）使用了正确标签，提供部分正确监督

**解决方法**:
```python
# 正确代码
tgt_onehot = tgt[batch_mask]
tgt_cls = tgt_onehot.argmax(dim=1).long()
```

**效果**:
- ✅ 已修复

---

### 4.2 JRC Water no-data (-128) 被错误参与 Reconstruction

**问题描述**:
- JRC Water 中 `-128` 是 no-data（占比 30-40%）
- 被当作有效值做 z-score：`-128 → -1.26`
- NaN 在 bilinear resize 中灾难性扩散

**解决方法**:
- 在 read_tif 后（43×43）、normalize 前设置 NaN
- resize 时用 nearest 避免扩散
- stats 重新计算，排除 -128

**效果**:
- ✅ JRC Water 重建更干净，~1/3 无效监督信号被移除

---

### 4.3 下游评估中的维度不匹配

**问题描述**:
- embedding_map=[128,64,64]，mask=128×128 → 维度不匹配直接 crash
- JRC Water 43×43 上采样到 128×128 后又需下采样到 64×64

**解决方法**:
- 所有 mask 统一下采样到 64×64（最近邻保持标签）
- JRC Water 直接上采样到 64×64

**效果**:
- ✅ 已修复

---

### 4.4 变化检测正负样本极端不平衡

**问题描述**:
- 正负比 884:1
- 模型倾向于全预测 0
- AUC/IoU 虚高

**解决方法**:
- BCE Loss 添加 `pos_weight=total_neg/total_pos`
- Focal Loss + Dice Loss

**效果**:
- ✅ 缓解类别不平衡

### 4.5 Shapefile CRS 未正确处理

**问题描述**:
- 哈尔滨 shapefile 原始 CRS 为 `None`
- 原代码直接跳过重投影
- 标注映射错误

**影响**:
- Few-Shot CD AUC 从 ~0.65 → ~0.50

**解决方法**:
```python
if gdf.crs is None:
    gdf = gdf.set_crs("EPSG:4326")
if gdf.crs.to_epsg() != 32652:
    gdf = gdf.to_crs(epsg=32652)
```

**效果**:
- ✅ AUC 恢复正常

### 4.6 matplotlib `imshow(origin='lower')` 导致图像上下翻转

**问题描述**:
- `rasterio.read()`: row=0 对应影像最北端
- `imshow(origin='lower')`: row=0 显示在底部 → 与地理坐标相反

**影响**:
- 标注叠加卫星图像上下颠倒

**解决方法**:
- 使用默认 `origin='upper'`

**效果**:
- ✅ 已修复

### 4.7 分类能力与变化检测能力的 Trade-off

**问题描述**:
- 分类能力 和 变化检测能力 不是正相关，甚至负相关
- 让 embedding "更聚类"（提升分类）→ 变化前后差异变小 → AUC 下降
- 让 embedding "更分散"（提升变化检测）→ 类别边界模糊 → 分类下降

**数据证据** (Round 4):

| 实验 | AUC | WC MLP | DW MLP |
|------|-----|--------|--------|
| baseline | **0.6918** | 43.8% | 50.7% |
| high_consist | 0.6509 (-0.041) | **47.4%** | 51.8% |
| high_temporal | 0.6757 (-0.016) | 47.0% | **53.9%** |

**解决方法**:
- 高 kappa (κ=5000) 是目前唯一"两全其美"的参数

**效果**:
- high_kappa: AUC=0.6844 (-0.007)，下游分类全面提升

## 七、经验教训汇总

### 7.1 关于反坍缩

1. **GMP 是坍缩的第一加速器** — 在 GMP 之后计算任何 uniformity/VICReg 都无法阻止坍缩
2. **Spatial VICReg 是唯一有效的反坍缩机制** — 在 `[B×H×W, D]` 上计算，保留空间多样性
3. **Reconstruction 权重是核心杠杆** — recon=0.5 导致坍缩，recon=0.1 是平衡点
4. **Raw Uniformity 在 pre-norm 空间有害** — 反而加剧坍缩
5. **Skip L2 是必要的但不充分** — 必须配合 Spatial VICReg

### 7.2 关于时序敏感性

1. **同月自重建 = 时间盲** — 必须引入跨时相重建或强时序对比
2. **Temporal Loss 权重过高会削弱 AUC** — 需要找到 sweet spot
3. **Pixel-level 时序优于 global mean** — 避免全局偏移作弊
4. **L2 norm 在推理时会压缩 pre-norm 的时序信号** — 需要验证 pre-norm AUC
5. **数据量小 → 模型忽略时间** — 这是根本限制

### 7.3 关于评估

1. **Bare AUC 是唯一诚实的指标** — CD Head AUC 可能包含虚假相关
2. **标注时间窗口必须匹配数据时间范围** — 否则 AUC 接近随机
3. **分类能力和变化检测能力存在 trade-off** — 不能同时最大化
4. **KNN 和 MLP 差异大** — MLP 更能反映 embedding 的真实质量

### 7.4 关于工程
